¿Qué es una branch?

  • Una branch (rama) es una línea de desarrollo independiente.
  • Permite trabajar en diferentes características o correcciones de bugs de manera aislada.
  • git checkout <nombre_de_la_rama>: Cambia a la rama especificada.
  • git switch <nombre_de_la_rama>: Cambia a la rama especificada (más moderno y seguro).

checkout

Un comando versátil utilizado para cambiar de rama o restaurar archivos en tu directorio de trabajo.

Permite navegar entre ramas, crear nuevas ramas, y revertir archivos o commits específicos.

Blame y Cherry-Pick

git blame: Muestra la última modificación de cada línea en un archivo.

Uso Común: Identificar quién hizo un cambio específico y cuándo.

git cherry-pick: Aplica los cambios de un commit específico a la rama actual.

Bisect

  • git bisect start: inicia el proceso
  • git bisect bad <commit_malo>: marca un commit como malo
  • git bisect good <commit_bueno>: marca un commit como bueno

Readme.md

Es un archivo de texto (generalmente en formato Markdown) ubicado en la raíz del proyecto.

  • Introducción al Proyecto: Explica qué es el proyecto, su propósito y objetivos.
  • Instrucciones de Instalación: Guía sobre cómo instalar y configurar el proyecto.
  • Uso del Proyecto: Proporciona ejemplos y detalles sobre cómo usar la aplicación o librería.
  • Licencia: Información sobre la licencia bajo la cual se distribuye el proyecto.

pre-commit

Configuración Básica:

  • Instalación: pip install pre-commit
  • Crear un archivo .pre-commit-config.yaml y definir hooks.
  • Instalar hooks en el repositorio: pre-commit install

Propósito:

  • Asegurar que el código cumpla con los estándares definidos.
  • Prevenir errores comunes al aplicar verificaciones automáticas.

hooks

Scripts que se ejecutan automáticamente en momentos específicos del ciclo de vida de Git.

Se utilizan para automatizar tareas, verificar condiciones antes de hacer commit, push, entre otras.
